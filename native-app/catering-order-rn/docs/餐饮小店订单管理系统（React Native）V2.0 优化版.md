# 一、文档总则

## 1.1 开发背景

本项目为个人独立开发、纯本地化离线餐饮订单管理 APP，删除原有运维管理工具，仅保留商家端 Android 应用。基于 React Native 轻量化开发，适配安卓 7.0 及以上低端机型（2026 年存量仍较高的入门级安卓设备），无联网请求、无云端存储，全部数据本地留存，满足小型餐饮门店接单、库存、统计、账目管理核心需求。

## 1.3 架构设计原则（新增性能 / 安全维度）

1. **易用性**：分层解耦，代码低耦合，个人开发维护成本低；

2. **扩展性**：预留打印、收款、硬件适配接口，后期无需重构架构；

3. **复用性**：封装通用原生模块、公共 UI 组件、工具类，全局复用；

4. **稳定性**：数据库事务兜底、归档容错、异常捕获，防止闪退数据错乱；

5. **离线化**：全部功能无网络依赖，禁止任何主动联网代码；

6. **性能优先**：所有模块优先适配低端机算力 / 内存限制，无过度动画 / 渲染；

7. **安全强化**：本地数据加密存储、权限最小化、操作日志留痕，符合 2026 年安卓隐私规范。

# 二、整体技术架构（分层架构）

## 2.1 架构分层结构图

从上层至下层分为：UI 展示层 → 业务逻辑层 → 数据管理层 → 原生桥接层 → 安全适配层（新增），严格分层，单向依赖。

> 优化说明：新增安全适配层，统一处理数据加密、权限校验、操作日志，适配 2026 年安卓隐私要求。

## 2.2 各层级职责说明（补充优化）

### 2.2.1 UI 展示层

负责所有页面渲染、用户交互、弹窗提示、动画效果，无业务逻辑，仅做页面展示与事件派发。包含页面组件、基于**Gluestack UI**封装通用UI组件、自定义弹窗、Tab栏、侧边栏。

> 优化补充：全部UI组件基于Gluestack静态编译，杜绝运行时重渲染；所有列表组件强制开启虚拟列表（VirtualizedList），图片组件内置压缩/懒加载，动画仅保留核心反馈（如加购），禁用过度动效；统一关闭Gluestack多余阴影、渐变特效，适配低端GPU。

### 2.2.2 业务逻辑层

核心业务处理，拆分独立业务模块，管控订单、菜品、库存、归档、权限逻辑，与 UI 层解耦，支持复用。包含订单服务、菜品服务、统计服务、权限服务、归档服务。

> 优化补充：新增 “操作日志服务”，记录敏感操作（退款、删除菜品），本地加密存储，便于问题追溯；业务方法增加低端机耗时阈值限制（单次操作≤500ms）。

### 2.2.3 数据管理层

管控本地 SQLite 数据库、数据缓存、数据备份、数据库迁移。封装通用数据库工具类，统一增删改查语法，支持 Migration 版本迁移。

> 优化补充：新增数据库索引（订单号、日期、菜品 ID），提升查询性能；备份数据采用 AES-256 加密，适配安卓 7.0 + 加密算法；缓存采用 LRU 策略，限制最大占用内存≤100MB。

### 2.2.4 原生桥接层

对接安卓原生能力，通过 Native Module/TSI（Turbo Native Modules）桥接离线语音、生物识别、文件读写、设备权限，屏蔽原生系统差异，JS 层直接调用封装方法。

> 优化补充：2026 年优先使用 TSI 替代老旧 Native Module，提升 JS - 原生通信效率；原生代码兼容安卓 7.0-15（2026 年最新版），屏蔽系统 API 差异。

### 2.2.5 安全适配层（新增）

统一处理数据加密 / 解密、权限申请与兼容、操作日志、生物识别校验、密码哈希存储，向下兼容安卓 7.0 安全机制，向上适配安卓 13 + 隐私权限（如文件访问、生物识别授权）。

# 三、全部页面规划、功能及交互设计（性能优化）

## 3.1 页面整体清单

APP 共 12 个页面，分为：初始化流程页面、通用页面、接单模式页面、管理模式页面，无冗余页面，适配极简操作。

> 优化补充：所有页面开启 “预加载 + 懒渲染”，非首屏页面延迟加载，降低启动耗时（目标≤3 秒）。

## 3.2 页面详细设计（功能 + 交互）（补充性能 / 安全优化）

### 3.2.1 初始化流程页面

#### 1、初始化配置页

- **页面功能**：首次安装强制初始化，填写店铺名称、店铺桌子数量、管理员密码、上传收款码，无跳过按钮；初始化完成写入系统配置表。

- **页面交互**：输入框非空校验，密码二次确认；提交按钮置灰防重复点击；配置完成自动跳转新手引导页。

> 优化补充：密码输入采用Gluestack原生受控输入框，关闭美化特效，避免自定义组件卡顿；收款码图片自动压缩至200KB以内，适配低端机存储；全部表单组件关闭实时校验防抖，减少CPU占用。

#### 2、新手引导页

- **页面功能**：蒙层沉浸式引导，覆盖核心操作流程；支持手动滑动切换引导步骤。

- **页面交互**：首次启动强制查看；引导完成关闭蒙层；侧边菜单可手动唤起复看。

> 优化补充：引导页采用Gluestack平铺View渲染，无透明蒙层过度渲染，滑动帧率≥30fps（适配低端机），支持一键跳过（仅管理员权限）。

#### 3、免责声明弹窗页

- **页面功能**：展示个人开发、数据免责、非商用声明；勾选同意方可进入 APP。

- **页面交互**：首次打开强制弹窗，无关闭按钮；勾选同意后永久不再弹窗，可在关于页面复看。

> 优化补充：声明内容采用Gluestack原生文本控件渲染，固定行高、关闭自适应排版，适配安卓7.0+文本渲染，避免乱码、排版错乱。

### 3.2.2 通用公共页面

#### 1、侧边抽屉菜单

- **页面功能**：模式切换、新手教程、关于我们、退出应用。

- **页面交互**：左上角图标侧滑唤起；接单切管理触发密码 / 生物识别校验；管理模式无操作超时自动关闭抽屉回落接单页。

> 优化补充：侧边菜单基于Gluestack Drawer封装，删减高斯模糊、动画插值，侧滑唤起帧率≥30fps，超时时间可配置（默认300秒），回落前增加Toast提示。

#### 2、密码验证页

- **页面功能**：管理模式身份校验，支持密码 + 生物识别双重验证；内置密钥本地重置密码。

- **页面交互**：密码错误 toast 提示；生物识别失败弹窗提示；忘记密码跳转重置流程。

> 优化补充：密码错误次数限制（5次锁定5分钟），全部按钮、输入框基于Gluestack封装，触控热区≥48dp；重置密码需验证设备本地存储的密钥（AES加密），生物识别失败后自动降级为密码验证。

### 3.2.3 接单模式页面（前台）

#### 1、简报统计 Tab 页

- **页面功能**：展示当日订单数、营收金额、热销菜品；无复杂图表，极简数据展示。

- **页面交互**：实时刷新当日订单数据；点击数据无跳转，仅静态展示；下拉刷新手动同步最新数据。

> 优化补充：数据刷新采用 “增量更新”，仅更新变化字段，降低数据库查询耗时；下拉刷新采用Gluestack极简Loading，删减动画曲线，避免低端机卡顿。

#### 2、点菜 Tab 页（核心高频页）

- **页面功能**：菜品展示、分类筛选、双购物车、语音点单、菜品加购。

- **页面交互**：
  1. 售罄菜品自动置灰、不可点击；

- **页面功能**：展示当日全部订单、筛选订单状态、查看订单详情、手动确认收款。

# 四、数据表详细结构设计（支持 Migration 迁移）

数据库名称：shop_local.db，采用**增量迁移 + 回滚方案**，版本号从 V1 开始递增，所有修改编写迁移脚本，事务执行、异常回滚，不删除历史数据。

## 4.1 通用数据库规范（优化 / 新增）

1. 主键统一自增 id，字段采用下划线命名；

2. 所有表增设创建时间（create_time）、更新时间（update_time）字段，类型为 TIMESTAMP（替代 TEXT，提升查询效率）；

3. 逻辑删除字段 is_delete，禁止物理删除业务数据；

4. 归档表人工不可修改，仅系统后台定时写入；

5. 核心业务表（订单 / 菜品）新增索引（如 order_table 的 order_no、create_time，dish_table 的 dish_name、is_sold_out_today）；

6. 敏感字段（如 admin_pwd）存储哈希值（Argon2），禁止明文存储；

7. 迁移脚本新增 “版本校验 + 回滚逻辑”，避免迁移失败导致数据库损坏。
