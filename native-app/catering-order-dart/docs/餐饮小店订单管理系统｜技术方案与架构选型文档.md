# 餐饮小店订单管理系统｜技术方案与架构选型文档
## 1 文档概述
本文档为**线下小型餐饮门店轻量化离线订单管理系统**的正式技术方案，明确系统整体架构、技术选型、模块设计、部署规范、安全策略与迭代规划，支撑个人开发者完成系统开发、交付与私有化运维，满足**纯本地部署、离线可用、数据安全、极简运维**的核心需求。

系统采用**双产品隔离体系**，由**商家端Android APP**（门店交付）与**私有化运维管理工具**（服务商自用）组成，全程无公有云、无联网采集、无商业化上架，实现数据主权与低成本维护。

## 2 整体设计原则
1. **数据主权优先**：门店经营数据100%留存本地设备，服务商不采集、不上传、不备份
2. **离线全程可用**：核心功能无网络依赖，断网可完整运行
3. **架构严格隔离**：数据、网络、部署、权限四层物理/逻辑隔离
4. **技术栈统一**：全栈Dart语言，降低个人开发与维护成本
5. **零成本运维**：无服务器费用、无付费SDK、无环境依赖
6. **业务安全可控**：订单不可篡改、权限分级管控、数据可溯源

## 3 核心架构选型
### 3.1 全栈技术选型（统一Dart）
| 系统模块 | 技术选型 | 选型核心依据 |
| ---- | ---- | ---- |
| 商家端Android APP | Flutter Android | 单端打包、轻量化渲染、适配低端安卓设备、开发效率高 |
| 运维工具前端 | Flutter Web | 编译静态资源、浏览器直接访问、无服务容器依赖 |
| 运维工具服务端 | Dart Server | 原生Dart、可编译Windows独立exe、裸机直接运行 |
| 本地数据库 | Sqlite + Drift | 单文件本地存储、Dart原生ORM、支持事务与增量迁移 |
| 离线语音能力 | 离线TTS + 离线ASR | 无联网请求、本地识别播报、适配口语化点单 |
| 数据导出 | 本地Excel生成 | 设备本地存储、无云端同步、满足对账需求 |

### 3.2 整体架构分层
1. **接入层**：商家端（Android移动端）、运维端（内网Web浏览器）
2. **业务逻辑层**：商家端业务（点单/订单/库存/统计）、运维端业务（台账/版本/产物管理）
3. **数据存储层**：商家本地Sqlite库、运维本地Sqlite库（物理隔离）

### 3.3 核心隔离设计
1. **数据隔离**：商家库存储经营数据，运维库仅存运维台账，无敏感数据互通
2. **网络隔离**：商家APP无主动联网，运维工具仅内网通信，不暴露公网
3. **部署隔离**：商家端部署于门店安卓设备，运维端部署于开发者Windows电脑
4. **权限隔离**：接单模式限前台操作，管理模式需鉴权，运维工具仅服务商自用

## 4 双产品体系技术架构
### 4.1 商家端Android APP 技术架构
#### 4.1.1 应用架构模式
采用**MVVM轻量架构**，适配Flutter生态，单人可快速维护：
- **表示层**：页面组件、双模式UI、交互入口、语音交互
- **领域层**：订单流转、库存管控、数据归档、权限校验、业务规则
- **数据层**：Drift数据库操作、本地数据读写、存储逻辑封装
- **工具层**：离线语音、Excel导出、定时任务、加密、容错处理

#### 4.1.2 核心技术能力
- 单APP双模式：接单/管理一键切换，权限动态管控
- 本地数据归档：定时触发+补偿触发，防重锁+事务兜底
- 离线语音体系：离线播报+离线识别，支持口语模糊匹配
- 订单风控：正向流转、管理员专属退款、数据不可篡改
- 数据库迁移：Drift增量迁移，升级不丢失数据

#### 4.1.3 存储架构
- 存储载体：Sqlite单文件本地存储
- 表结构：4张基础业务表 + 2张归档统计表 + 1张归档锁表
- 存储规则：原始流水不可修改、归档数据自动封存、逻辑删除保障数据链路

### 4.2 私有化运维管理工具 技术架构
#### 4.2.1 应用架构模式
采用**内网闭环前后端分离架构**：
- **前端**：Flutter Web静态资源，无跨域、无外部依赖
- **服务端**：Dart Server轻量化HTTP服务，编译为Windows独立exe
- **数据层**：Sqlite单文件本地库，存储运维台账数据

#### 4.2.2 核心技术能力
- 零部署成本：无需数据库、无需运行时、双击启动
- 版本管控：门店版本标记、APK产物归档、拦截降级安装
- 运维台账：门店信息、维护记录、数据库变更脚本管理
- 内网安全：仅本地/内网访问，无公网暴露风险

## 5 数据存储与迁移方案
### 5.1 数据库选型
- 统一使用**Sqlite**本地单文件数据库，无服务端依赖
- 统一使用**Drift**作为Dart ORM，支撑事务、查询、数据迁移

### 5.2 数据迁移规范
- 增量迁移：版本迭代仅新增表/字段，不删除、不修改历史数据
- 事务兜底：迁移异常自动回滚，保障数据库完整性
- 版本锁：数据库版本号递增，系统拦截降级安装

### 5.3 数据归档机制
- 触发规则：每日00:00定时执行 + 应用启动补偿执行
- 防重机制：本地时间锁表，避免重复归档与数据错乱
- 容错机制：事务包裹执行，异常自动回滚，保证统计数据准确

## 6 部署与分发技术方案
### 6.1 商家端部署规范
- 安装方式：Android离线APK人工线下分发，本地直接安装
- 升级方式：覆盖安装升级，Drift自动完成数据迁移
- 约束规则：禁止降级安装，系统弹窗拦截，防止数据库异常

### 6.2 运维端部署规范（Windows专属）
- 编译输出：服务端编译为exe可执行文件，前端编译为静态Web资源
- 启动方式：双击exe启动本地服务，浏览器通过内网地址访问
- 停止方式：关闭进程即停止服务，无后台驻留、无系统残留

## 7 数据安全与业务风控方案
### 7.1 数据安全方案
- 无联网上传：商家APP禁用网络权限，无任何数据上报行为
- 本地存储：经营数据仅留存门店设备，服务商无备份、无获取通道
- 密码安全：管理员密码加密存储，支持本地安全密钥重置

### 7.2 业务风控方案
- 权限风控：管理模式需密码/生物识别校验，闲置超时自动退出
- 订单风控：接单模式仅支持正向流转，管理模式才可执行退款
- 库存风控：下单自动扣减库存，售罄自动置灰，前台不可恢复售卖
