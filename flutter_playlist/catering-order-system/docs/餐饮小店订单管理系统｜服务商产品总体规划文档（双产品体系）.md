# 餐饮小店订单管理系统｜服务商产品总体规划文档（双产品体系）

# 一、服务商业务总览

## 1\.1 业务定位

本人为**独立个人开发者（服务商）**，面向线下小型餐饮门店，提供轻量化、低成本、纯本地化部署的餐饮订单管理解决方案。整体业务包含两大核心产品实体：面向终端门店的**商家移动端APP**、面向开发者自用的**私有化运维管理工具**。整套体系无公有云服务器、无联网采集、无商业化上架，主打离线可用、数据安全、极简运维、长期低成本迭代。

## 1\.2 核心产品实体划分

本方案由两款独立产品组成，数据严格隔离、职责明确，形成「交付产品\+运维管控」的完整服务体系：

1. **商家端Android APP（终端交付产品）**：交付给线下餐饮门店使用，单应用内置接单/管理双工作模式，承担门店点餐、订单流转、菜品维护、经营统计、本地数据存储等全部业务能力，全程离线运行。

2. **私有化运维管理工具（服务商自用）**：基于Flutter Web \+ Dart Server搭建的内网管理平台，仅开发者本机/内网访问，用于管控门店台账、应用版本、维护记录、编译产物管理，仅留存运维台账，不触碰任何门店经营隐私数据。

## 1\.3 整体业务约束（服务商准则）

- **开发属性**：个人独立开发，非商用发行软件，无官方运营团队。

- **上架约束**：所有产品永不上架应用市场，仅线下人工分发部署。

- **数据准则**：门店经营数据100%留存门店本地设备，不上传、不采集、不备份；服务商仅留存运维台账数据。

- **部署方式**：无公网服务器、无云端数据库，全部采用本地私有化部署。

- **成本控制**：全程零服务器费用、零付费SDK、零广告、无内置扣费，极致轻量化低成本。

# 二、总体技术架构（统一Dart技术栈）

## 2\.1 技术栈规划

整套产品体系**全部基于Dart语言开发**，技术栈高度统一，降低个人开发、迭代、维护成本，无额外学习成本：

1. **商家APP**：Flutter Android，单端打包，移动端适配。

2. **运维Web端**：Flutter Web，编译静态资源，浏览器直接访问。

3. **运维服务端**：Dart Server，轻量化本地服务，Windows平台部署。

4. **统一数据库**：Sqlite \+ Drift，本地数据库，支持数据库迁移。

## 2\.2 架构隔离设计

- **业务数据隔离**：商家APP存储订单、菜品、营收等经营数据；运维工具仅存储门店信息、版本记录、维护台账，无任何经营敏感数据。

- **网络隔离**：商家APP全程断网可用，无任何主动联网请求；运维工具仅内网本地通信，不暴露公网端口。

- **部署隔离**：商家APP部署于门店安卓手机；运维工具仅部署在开发者自用Windows电脑。

# 三、商家端Android APP（面向门店终端）

## 3\.1 产品定位

面向无专职服务员的小型餐饮门店，实现**点单、记账、订单管控、数据统计、库存管理**一体化离线工具。采用单APP双模式设计，无需下载多个安装包，一键切换工作模式，适配店主单人操作、口头报单、线下收款的经营场景。

## 3\.2 核心约束

- 纯本地存储，无云端同步，所有数据永久留存设备本地。

- 打印、线上收款为UI占位模块，暂不实现底层逻辑，预留后期扩展接口。

- 内置离线语音播报、离线语音识别，无需联网，适配快速点单。

- 依托Drift实现数据库版本迁移，保障APP迭代升级数据不丢失。

## 3\.3 数据架构（六表本地数据库）

严格区分「原始业务流水表」与「归档统计表」，降低低端安卓设备算力压力，保障长期运行流畅，所有归档数据人工不可修改，保证账目严谨性。

### 3\.3\.1 基础业务表

1. **菜品表**：存储菜品名称、售价、图片、上下架状态、库存配置、售罄标记。

2. **订单表**：存储订单编号、流转状态、金额、退款标记、时间记录。

3. **订单项表**：关联订单与菜品，记录下单数量、单品单价。

4. **系统配置表**：存储店铺信息、管理员密码、生物识别、收款码等系统配置。

### 3\.3\.2 归档统计表（系统自动生成）

1. **每日聚合统计表**：归档每日营收、订单、退款、客单价等总账数据，用于周期营收分析。

2. **菜品日统计表**：归档每日单品销量、营收、退款数据，用于菜品结构分析。

### 3\.3\.3 定时归档容错机制

- 常规触发：每日凌晨00:00静默执行库存重置、数据归档、数据封存。

- 补偿触发：若凌晨应用未启动，下次打开APP自动补执行归档任务。

- 防重锁机制：写入本地时间锁，防止当日重复归档、数据错乱。

- 事务兜底：归档异常自动回滚，保障统计表数据完整。

## 3\.5 接单模式（前台营业操作）

- Dart Server：开发环境需SDK，生产环境编译为exe，**无需任何运行时**，裸机可直接启动。
